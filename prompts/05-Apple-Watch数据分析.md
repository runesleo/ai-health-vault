# Prompt: Apple Watch / 健康数据分析

## 导出步骤

1. iPhone → 健康 App → 右上角头像 → 拉到最底部 → **导出所有健康数据**
2. 等待生成（可能需要几分钟）
3. 得到 `export.zip`（压缩后几十 MB ~ 几百 MB）
4. 传到电脑（AirDrop / 微信文件传输助手 / 数据线）

## 重要提示

**不要把原始 XML 直接丢给 AI。** 解压后的 XML 文件可能超过 1GB，太大太冗余。

正确做法：

```
我有一个 Apple Health 导出的 export.zip 文件，在 [文件路径]。
请帮我：
1. 解压并分析 XML 数据结构
2. 按类型提取关键数据（心率、步数、睡眠、运动、血氧等）
3. 转换为 CSV 格式方便后续分析
4. 生成完整的健康分析报告
```

优先做法：先运行仓库内脚本预处理。

```bash
python scripts/apple_health_preprocess.py --input /path/to/export.zip --output ./apple-health-csv
```

脚本会按常用数据类型拆分出独立 CSV（如心率、步数、睡眠、运动）。
你可以把这些 CSV 分批发给 AI 做进一步分析。

如果你当前环境没法运行脚本，再让 AI 现写一个兜底脚本：

```
这个 XML 文件太大了，请先帮我写一个 Python 脚本：
1. 解析 export.xml
2. 按数据类型（心率、步数、睡眠等）拆分成独立 CSV 文件
3. 每个 CSV 只保留：日期、时间、数值
这样我可以分批分析。
```

## 分析 Prompt

```
请基于导出的 Apple Watch 数据，分析以下维度：

### 心血管
- 静息心率趋势（月均）
- HRV 趋势（月均，标注是否下降）
- VO2Max 趋势（如有）
- 血氧趋势
- 低心率/高心率事件统计
- 心率分区分布（休息/有氧/燃脂/高强度各占比）

### 运动
- 运动类型和次数统计
- 近期运动模式变化（跟半年前对比）
- 日均步数趋势（按年/按月）
- 工作日 vs 周末步数差异

### 睡眠
- 睡眠时长趋势
- 深睡/REM/核心睡眠占比
- 入睡时间规律性

### 综合评价
- 好的方面（继续保持）
- 需要关注的问题（按优先级排序）
- 具体的改善建议（可执行的，不要泛泛而谈）

输出为 Markdown 格式，我要写入体检档案。
```

## 适用范围

- Apple Watch（全系列）
- iPhone 健康 App 记录的所有数据（含第三方 App 写入的数据）
- 安卓用户：Google Fit 导出后格式不同，需要调整脚本
