# 家庭友好版健康信息

将专业的健康分析结果转换为通俗易懂的微信消息格式，适合发给家人。

## 核心任务

把复杂的体检报告分析、医学建议等转换成亲和的家庭沟通语言，让没有医学背景的长辈也能轻松理解。

## 转换原则

### 语言风格
- **口语化表达**：像面对面聊天一样，避免书面语
- **简化专业术语**：必须使用时加括号解释
- **温和但直接**：不制造恐慌，但也不隐瞒重要信息
- **具体化建议**：给出可执行的行动，避免抽象概念

### 结构要求
- **控制长度**：微信一屏能看完，不超过200字
- **重点突出**：用换行和简单符号标注关键信息
- **行动明确**：如需就医、复查，明确写时间地点
- **情感收尾**：以关心话语结束

## 转换模板

### 基本结构
```
[称呼]，[检查结果概述]：

[正常项目简述]

[需关注项目]：
- [具体数值+通俗解释]
- [生活建议]

[需要做的事]：
- [具体行动+时间]

[关心话语] ❤️
```

### 专业术语转换表

| 医学术语 | 家庭版表达 |
|----------|-----------|
| 空腹血糖偏高 | 血糖有点高 |
| 低密度脂蛋白胆固醇 | 坏胆固醇 |
| 甲状腺功能亢进 | 甲状腺太活跃了 |
| 维生素D缺乏 | 缺维生素D（跟晒太阳少有关系） |
| 肝功能异常 | 肝脏指标不太好 |
| 建议复查 | 下次再检查一下 |

## 情景化处理

### 📊 体检报告解读
**重点**：
- 先说正常的，再说需要注意的
- 数值变化用"比上次高了一点"等相对表达
- 给出具体的生活调整建议

### 💊 用药指导
**重点**：
- 明确用药时间和方式
- 解释药物作用（治什么的）
- 提醒注意事项和副作用

### 🏥 就医建议
**重点**：
- 解释为什么要去看医生
- 明确科室和大概时间
- 缓解焦虑，说明问题不严重

## 示例转换

### 原始专业版
```
体检报告显示：空腹血糖6.3mmol/L，较上次5.8mmol/L有所升高，
虽仍在正常范围内但需要关注。维生素D水平18ng/mL，低于正常值。
建议6个月后复查血糖，同时补充维生素D。
```

### 家庭友好版
```
妈，上次体检报告我帮你看了一下：

血糖 6.3，比上次高了一点点，还在正常范围但要注意了。少吃甜的，米饭也别吃太多。

维生素D 偏低，这个跟晒太阳少有关系。可以每天出去走走晒半小时太阳，或者吃维生素D的补充剂（药店有卖的，一天一粒就行）。

下次复查建议 6 个月后再查一次血糖，到时候我提醒你。

注意身体，有不舒服随时跟我说 ❤️
```

## 特殊处理

### 异常结果
- 不用"异常"、"不正常"等刺眼词汇
- 改用"有点高/低"、"需要注意"等温和表达
- 强调可以通过调整改善

### 紧急情况
- 如需立即就医，语气坚定但不恐慌
- 解释紧急的原因但不过度详述风险
- 提供陪同就医的安排

### 长期管理
- 将复杂的管理方案简化为几个要点
- 用"每天做什么"的形式列出
- 避免一次性给太多建议

## 触发场景

- 用户要求将分析结果"发给我爸/妈"
- 用户询问"怎么跟家人解释这个报告"
- 用户需要帮长辈理解医生的建议
- 体检、就医结果需要传达给家人
