# Apple Watch 健康数据分析

处理和分析 Apple Health 导出的健康数据，生成综合健康分析报告。

## 核心任务

从 Apple Health 导出的大型 XML 文件中提取、分析健康数据，生成可操作的健康洞察。

## 数据处理流程

### 第一步：数据预处理

⚠️ **重要**：绝不直接处理原始 XML 文件（通常 >1GB）

当用户提供 export.zip 时：
1. 优先使用仓库内脚本 `scripts/apple_health_preprocess.py`
2. 将 XML 按类型拆分为 CSV
3. 再按数据类型分批处理
4. 如果用户当前环境无法运行脚本，再临时提供 Python 兜底脚本

### 第二步：数据分类提取

**心血管数据**
- 心率（静息、运动、恢复）
- 心率变异性(HRV)
- VO2Max
- 血氧饱和度
- 心率分区分布

**运动数据**
- 步数统计
- 运动类型和频次
- 卡路里消耗
- 站立时间

**睡眠数据**
- 睡眠时长
- 睡眠阶段（深睡/REM/核心睡眠）
- 入睡时间规律性

## 分析维度

### 🫀 心血管健康

**静息心率趋势**
- 月均值变化
- 与年龄标准对比
- 识别异常升高/降低

**HRV 分析**
- 长期趋势（是否下降警示）
- 压力和恢复指标
- 训练状态评估

**心率分区分析**
- 休息区间占比
- 有氧运动区间
- 高强度区间
- 运动效率评估

### 🏃‍♀️ 运动表现

**活动量趋势**
- 日均步数变化
- 工作日 vs 周末差异
- 季节性变化模式

**运动模式分析**
- 运动类型偏好变化
- 运动频次和强度趋势
- 与半年前/一年前对比

### 😴 睡眠质量

**睡眠时长趋势**
- 平均睡眠时长变化
- 睡眠债累积情况

**睡眠质量指标**
- 深睡占比变化
- REM 睡眠占比
- 入睡时间规律性

**睡眠模式**
- 就寝时间一致性
- 周末补觉模式
- 睡眠效率变化

## 输出格式

### 📊 数据总览表

| 指标 | 近期平均 | 6个月前 | 变化趋势 | 评估 |
|------|----------|---------|----------|------|
| 静息心率 | 65 bpm | 68 bpm | ↓ 改善 | ✅ 良好 |
| HRV | 45 ms | 42 ms | ↑ 改善 | ✅ 良好 |

### 🎯 重点发现

**✅ 积极变化**
- [具体改善的指标和数据]
- 可能原因分析
- 建议继续保持的行为

**🔴 需要关注**
- [恶化或异常的指标]
- 可能风险评估
- 具体改善建议

**⚠️ 监测要点**
- [波动较大的指标]
- 建议关注的时间点

### 🔧 具体改善建议

**运动方面**
- 具体的步数目标调整
- 推荐的运动类型和频次
- 避免过度训练的信号

**睡眠方面**
- 睡眠时间优化建议
- 改善睡眠质量的具体方法

**心血管健康**
- 基于数据的有氧训练建议
- 压力管理建议

## 技术处理脚本

优先脚本：

```bash
python scripts/apple_health_preprocess.py --input /path/to/export.zip --output ./apple-health-csv
```

如果当前环境不能运行脚本，再提供临时 Python 兜底脚本。

## 兼容性说明

**适用设备**
- Apple Watch 全系列
- iPhone 健康 App 记录的所有数据
- 第三方 App 写入健康 App 的数据

**其他平台**
- 安卓用户：Google Fit 导出格式不同，需要调整脚本
- 华为/小米手环：需要相应的数据导出和转换方案

## 触发场景

- 用户提供 Apple Health 导出的 export.zip 文件
- 用户询问"分析我的运动数据"、"看看心率趋势"
- 用户想了解健康设备数据的含义
- 用户准备体检，想了解持续监测的健康状况
